输出文件夹

                    

                        

                            
                            
                        

                        
                        

                            

                                

                                    
作用

                                    
作为 Web 训练的统一输出根目录；每次启动训练或预处理时，系统会在这里创建独立运行目录，集中保存模型缓存、数据集缓存、训练结果和运行时配置快照。

                                

                                

                                    
为什么

                                    
把输出和缓存集中到同一个根目录，能避免配置页和数据集页反复填写缓存路径，也更容易按训练时间找到结果、复盘配置和清理旧运行。

                                    
PS：缓存必须要有地方存放；让用户逐项单独配置太繁琐，所以最终只指定一个主路径来集中存放缓存和结果，既让用户知道文件会放在哪里，也减少需要配置的项目。

                                

                            

                        

                    

                    
基础模型路径 (3 项)

                    

                        

                            
                            
                        

                        
                        

                            

                                

                                    
作用

                                    
作为新建空白预设配置时写入的默认基础 DiT 权重路径；配置页里的同名字段仍可单独覆盖。

                                

                                

                                    
为什么

                                    
把最常复用的底模路径放到全局设置，可以减少每次新建配置后重复改路径的步骤。

                                

                            

                        

                    

                    

                        

                            
                            
                        

                        
                        

                            

                                

                                    
作用

                                    
作为新建空白预设配置时写入的默认文本编码器权重路径；单个训练配置仍可改成其它路径。

                                

                                

                                    
为什么

                                    
文本编码器通常全项目共用，放到全局设置后，新预设能直接继承当前机器上的常用模型位置。

                                

                            

                        

                    

                    

                        

                            
                            
                        

                        
                        

                            

                                

                                    
作用

                                    
作为新建空白预设配置时写入的默认 VAE 权重路径；配置页里的 VAE 路径仍可按配置覆盖。

                                

                                

                                    
为什么

                                    
VAE 会影响预处理缓存和采样结果，集中设置默认路径能降低新配置漏改模型组件的概率。

                                

                            

                        

                    

                    

                        
                        
                    

                    

                

            

        

    


    
        

            

                
训练前预检测

                
            

            

            

            

                
                
                
            

        

    

    
        

            

                

                    
基础教程

                    
按这个顺序配置，先让 WebUI 能完成预处理并启动训练。

                

                
            

            

                

                    
训练前必须完成

                    
    
                        
  1. 
                            全局设置
                            设置“输出文件夹”，再填写基础 DiT、Qwen3、VAE 路径。之后每次 Web 训练都会在这里创建独立运行目录。
                        
    2. 
                        
  2. 
                            数据集
                            新建或选择数据集预设，只填原始数据集路径、重复次数和数据集参数；保存后回到配置页引用它。
                        
    3. 
                        
  3. 
                            配置
                            选择方法、变体、预设。新手优先使用 LoRA/default；确认模型路径、数据集配置、训练步数、采样提示词等关键字段。
                        
    4. 
                        
  4. 
                            保存配置
                            修改左侧表单后先保存。TOML 直接编辑也要保存，否则“开始训练”不会使用未写入磁盘的内容。
                        
    5. 
                        
  5. 
                            开始训练
                            点击“开始训练”后先看预检测。缺少缓存时 WebUI 会自动先预处理，完成后再进入训练。
                        
    6. 
                    

                

                

                    
跑起来后怎么看

                    

                        

                            训练页
                            看当前状态、日志、Loss 曲线和历史任务。从检查点恢复也在这里操作。
                        

                        

                            预览结果页
                            看训练中采样图、权重文件；权重可以直接通过浏览器下载。
                        

                        

                            输出目录
                            每次运行目录包含 config.original.toml、config.runtime.toml、dataset_cache、model_cache 和 training_output。
                        

                    

                

                

                    
常见卡点

                    
    
                        
  • 模型路径为空或指错时，先去“全局设置”修正，再回配置页使用“填写全局路径配置”。
    • 
                        
  • 数据集页只管理原始图片路径；缩放图和 LoRA 缓存会自动放进本次运行目录。
    • 
                        
  • 直接点训练也会先保证预处理链路；不要手动把旧缓存目录写回配置里。
